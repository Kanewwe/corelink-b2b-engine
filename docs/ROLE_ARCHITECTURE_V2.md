# Linkora B2B 引擎 — 角色架構重新定義

> **版本：** 2.0.0  
> **更新日期：** 2026-03-26  
> **核心變更：** Vendor 改為「簽約合作夥伴」，非「旗下會員管理」

---

## 一、三角色重新定義

### 1.1 Admin (平台管理員)

**定義：** Linkora 平台營運方

**權限：**
- 全系統管理與監控
- Vendor 開通與管理
- 方案定價設定 (免費/專業/企業)
- 系統參數配置 (API Keys)
- 全系統數據分析

**收費模式：**
- 向 Vendor 收取平台服務費 (抽成或月費)

---

### 1.2 Vendor (簽約合作夥伴) ⭐ 重新定義

**定義：** 與 Linkora **簽訂合作合約**的 B2B 探勘服務商

**特權：**
- ✅ **無限探勘** (leads 無上限)
- ✅ **無限郵件發送** (emails 無上限)
- ✅ 進階爬蟲模式 (製造商模式優先)
- ✅ API 存取權限
- ✅ 自定義郵件發送域名 (白牌)
- ✅ 優先技術支援

**限制：**
- ❌ 無法管理其他用戶
- ❌ 無法查看其他 Vendor 資料
- ❌ 需簽約並通過審核

**收費模式：**
- 與 Linkora **簽約月結**
- 計價維度：
    1. **單次發送 (Outreach)**：每封寄出信件之費用
    2. **有效觸及 (Reach)**：客戶開信/點擊之成效費用
    3. **訂單確定 (Conversion)**：最終成交/訂單確認之分潤或固價
- 自行向其客戶收費 (Linkora 不介入)

**典型客戶：**
- B2B 探勘顧問公司
- 外貿代操服務商
- 大型企業內部業務團隊

---

### 1.3 Member (一般會員) — 分級方案

**定義：** 一般 B2B 業務開發人員

**分級方案：**

| 方案 | 月費 (TWD) | leads 配額 | emails 配額 | templates | 功能限制 |
|------|-----------|-----------|------------|-----------|---------|
| **免費版** | $0 | 10 | 10 | 1 | 僅黃頁模式 |
| **專業版** | $899 | 100 | 100 | 5 | 黃頁 + 製造商 |
| **企業版** | $2,999 | 500 | 500 | 無限 | 全功能 + API |

**升級路徑：**
```
免費版 → 專業版 → 企業版 → Vendor (簽約)
```

## 三、業務流程重新設計

### 3.1 Vendor 開通流程

```
Step 1: 初步接觸
├─ 潛在 Vendor 透過官網或業務接洽
├─ 了解需求與使用場景
└─ 提供功能演示

Step 2: 資格審核
├─ 公司登記證明
├─ 業務規模評估
├─ 信用調查
└─ Admin 審核通過

Step 3: 合約簽訂
├─ 議價服務費用 (月費/年約/抽成)
├─ 簽訂合作合約
├─ 設定服務等級 (SLA)
└─ 開通 Vendor 帳號

Step 4: 技術整合
├─ 設定自定義發信域名
├─ API 金鑰開通
├─ 技術教育訓練
└─ 正式上線
```

### 3.2 Member 轉換 Vendor 路徑

```
Member (企業版) 使用 3 個月
    ↓ 用量穩定超標
申請升級 Vendor
    ↓
Admin 審核業務資格
    ↓
議價簽約
    ↓
開通 Vendor 帳號
    ↓
無限使用 + 自定義品牌
```

## 四、收費模式對照

### 4.1 平台收費 (Admin)

| 來源 | 收費方式 | 說明 |
|------|---------|------|
| Member 月費 | 方案訂閱 (Free/Pro/Ent) | 直接收入 |
| Vendor 服務費 | **三維度計價 (發送/觸及/成交)** | 依合約議定之單價結算 |
| 加值服務 | 按次/按量 | 額外 AI 額外、代客探勘等 |

### 4.2 Vendor 收費 (自行管理)

| 模式 | 說明 | 範例 |
|------|------|------|
| 專案服務費 | 按專案收費 | 幫客戶探勘 1000 leads，收費 $5000 |
| 月費訂閱 | 提供無限探勘服務 | 客戶月付 $500，無限使用 |
| 成效抽成 | 按成交金額抽成 | 成交 $100k，抽 5% = $5k |

## 六、技術實作調整

### 6.1 需要修改的部分

| 項目 | 原設計 | 新設計 |
|------|--------|--------|
| User.vendor_id | 指向管理他的 Vendor | 移除欄位 |
| Vendor 模型 | 管理旗下會員 | 僅存合約資訊 |
| 用量檢查 | 依 vendor_id 彙總 | 依 role + plan 檢查 |
| 對帳報表 | Vendor 旗下會員彙總 | 移除 (Vendor 自行管理) |

### 6.2 用量檢查邏輯

```python
def check_quota(user, action_type):
    """
    檢查用戶用量配額
    """
    if user.role == "vendor":
        # Vendor 無限使用
        return True, -1
    
    if user.role == "member":
        plan = user.subscription.plan
        usage = get_monthly_usage(user.id)
        
        if plan == "free":
            limit = FREE_LIMITS[action_type]
        elif plan == "pro":
            limit = PRO_LIMITS[action_type]
        elif plan == "enterprise":
            limit = ENTERPRISE_LIMITS[action_type]
        
        return usage < limit, limit - usage
    
    return False, 0
```
